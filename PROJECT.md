## 产品定位

面向熟悉或学习 **六爻纳甲** 的用户，提供 **在线摇卦、自动排盘、查看本卦/变卦、神煞与动静** 等的 Web 体验。  
**前端**：交互与展示；**后端（Spring Boot）**：装卦入库、排盘数据与业务规则。

## 用户流程

1. **起卦**：未登录时先完成登录（或注册后登录）→ 进入首页 → 可选填写所问（标题）→ **自下而上摇卦六次**（每次定一爻），或在 **高级 · 手动录入** 中为六爻选择四象 → 提交排盘 → 跳转结果页（携带记录 id）。
2. **结果**：查看四柱与卦象详情；返回首页或进入历史（接口就绪后）。
3. **用神（MVP，规划中）**：在结果页 **点击本卦某一爻** 选中为用神 → 弹出 **确认**（文案含爻位与六亲，避免误触）→ 用户确认后调用后端用神相关接口 → 在 **卦盘区域下方** 展示本次计算结果（文字或数值摘要以后端为准）。
4. **历史**（能力就绪后）：列表展示过往记录 → 点击条目进入对应结果页。
5. **登录与主站访问**：进入起卦、历史、结果等主流程前需已登录；登录/注册页与认证 API 可匿名访问；会话与路由守卫见 **architecture.md**（`middleware`、`token` cookie 与 `localStorage`）。

## 用神与后续迭代（产品意图）

- **对内统一用爻位**：前后端协议中的 `yongshen` / `yongshen_yao` 均指 **爻位编号**（初爻=1，上爻=6），与排盘数据里 `GuaInfo.shi` / `ying` 的编号方式一致；用户通过 **点选具体爻** 表达用神，无需单独「六亲下拉」消解多爻同六亲的歧义。
- **未来：月日干支试算与图表**：在不改「排盘为主视觉」的前提下，计划在结果页增加 **可折叠的高级参数**（如调整月、日干支），对同一卦、同一用神计算多情景指标，并在用神结果区用 **图表** 对比；具体字段与接口在 **architecture.md** 与 `草稿.md` 随契约定稿同步。
